Analisis Perbandingan Model Proses Bisnis

Studi Kasus: Implementasi Sistem Informasi Monitoring Kunjungan (SIMVIS)

Model Proses Bisnis Berjalan (As-Is) - Manual/Telegram

Analisis Masalah (As-Is):

  • Manual & Redundan: Pelaporan via chat Telegram mengharuskan rekapitulasi ulang secara manual.
  • Validasi Lemah: Tidak ada validasi lokasi (GPS) sistematis, hanya bergantung pada foto/shareloc.
  • Decision Making Lambat: Data tidak realtime, manajer harus menunggu rekapitulasi selesai.

Model Proses Bisnis Usulan (To-Be) - SIMVIS System

Analisis Solusi (To-Be):

  • Otomatisasi: Data langsung masuk database saat Sales update status.
  • Validasi Sistem: Sistem memvalidasi radius GPS secara otomatis (< 100m) sebelum Check-In.
  • Realtime: Dashboard manajer terupdate seketika, memungkinkan keputusan cepat.
